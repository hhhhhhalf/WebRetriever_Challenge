参赛流程总览

Participation Overview

1
报名 & 组队

在 Octo 平台注册账号，加入 WebRetriever 赛事空间，完成报名并组队。

2
阅读论文 & 熟悉开源项目

报名后即可进入备赛。建议先阅读 WebRetriever 论文，了解数据集中的三种评测协议（Protocol I / II / III），然后基于我们的开源代码仓库开始构建自己的 Agent——比赛代码模板在很大程度上也基于该开源项目设计。

3
获取队伍专属仓库

报名期结束后，我们会基于比赛代码模板为每个队伍创建一个私有 GitHub 仓库。提供了 GitHub 账号的队员将被添加为 Collaborator，可直接编辑仓库代码。届时将你在备赛期开发好的 Agent 迁移到队伍仓库即可。

4
提交 & 评测

将代码 push 到仓库 main 分支，通过赛事 Bot 发起评测，获取评分反馈。

1
Register & Form a Team

Create an Octo account, join the WebRetriever Competition Space, and complete registration with your team.

2
Read the Paper & Explore the Open-Source Project

Start preparing right after registration. Read the WebRetriever paper, understand the three evaluation protocols (Protocol I / II / III) in the dataset, then begin building your Agent based on our open-source code repository — the competition code template is largely designed around it.

3
Get Your Team Repository

After the registration period ends, we will create a private GitHub repository for each team based on the competition code template. Members who provided their GitHub accounts will be added as Collaborators with edit access. Simply migrate the Agent you developed during preparation into your team repository.

4
Submit & Evaluate

Push your code to the main branch, initiate evaluation through the Competition Bot, and receive your score.

比赛代码模板

Competition Code Template

比赛代码模板基于上述开源代码仓库设计，保留了 Agent 的核心执行框架。选手在模板基础上开发自己的 Agent 逻辑。

The competition code template is built upon the open-source repository above, retaining the core Agent execution framework. Participants develop their own Agent logic on top of this template.

选手需要关注的接口

Interfaces You Need to Know

模板已经预设好了大部分运行逻辑，选手只需关注以下几个关键点：

Most of the runtime logic is pre-configured in the template. You only need to focus on the following key points:

  • 输入文件 & 输出路径 — 评测系统会通过命令行参数传入任务文件路径和结果输出目录，模板已处理好参数解析，选手无需修改
  • CDP URLs（浏览器沙箱地址） — 评测系统会提供云端浏览器沙箱的 CDP 连接地址，选手的 Agent 通过这些地址连接浏览器执行任务
  • API Key 配置 — 选手需要在 config.json 中填写自己的模型 API 配置（API Key、Base URL 等），评测时使用选手自己的推理资源
  • Input File & Output Path — The evaluation system passes the task file path and result output directory via command-line arguments. The template handles argument parsing; no modification needed
  • CDP URLs (Browser Sandbox Addresses) — The evaluation system provides cloud browser sandbox CDP connection URLs. Your Agent connects to browsers through these URLs to execute tasks
  • API Key Configuration — You need to fill in your own model API configuration (API Key, Base URL, etc.) in config.json. Evaluation uses your own inference resources

💡 简而言之：输入文件、输出路径、浏览器沙箱由评测系统提供；模型推理资源（API Key）由选手自己准备。选手的核心任务是开发 Agent 的决策和执行逻辑。

💡 In short: Input files, output paths, and browser sandboxes are provided by the evaluation system; model inference resources (API Keys) are prepared by participants. Your core task is to develop the Agent's decision-making and execution logic.

评测方式

Evaluation Process

评测采用自动化流水线：

Evaluation uses an automated pipeline:

  1. 将代码 push 到队伍私有仓库的 main 分支
  2. 通过赛事 Bot 发起"提交评测"
  3. 系统自动拉取代码 → 静态检查 → 安装依赖环境 → 在云端沙箱中执行评测
  4. 评测完成后获得评分反馈
  1. Push your code to the main branch of your team's private repository
  2. Initiate "Submit for Evaluation" through the Competition Bot
  3. The system automatically pulls your code → runs static checks → installs dependencies → executes evaluation in cloud sandboxes
  4. Receive scoring feedback upon completion

📌 评测窗口期内允许多次提交，取最优成绩作为该阶段得分。评测技术细则将在备赛期详细公布。

📌 Multiple submissions are allowed within each evaluation window; the best score counts as the stage result. Detailed technical specifications will be released during the preparation period.

常见问题

FAQ

Q: 需要自己准备 GPU / 推理服务器吗？

Q: Do I need my own GPU / inference server?

A: 是的。参赛 Agent 通过 OpenAI 兼容接口调用模型，选手需自行管理模型推理资源。可以使用商业 API（如 OpenAI、Claude 等）或自部署模型。

A: Yes. Participating Agents call models via OpenAI-compatible APIs. Teams are responsible for their own model inference resources. You can use commercial APIs (OpenAI, Claude, etc.) or self-hosted models.

Q: 浏览器环境需要自己搭建吗？

Q: Do I need to set up the browser environment myself?

A: 不需要。评测时由平台提供云端浏览器沙箱（通过 CDP 协议连接），选手无需搭建浏览器环境。本地调试时可自行使用 Playwright 等工具。

A: No. The platform provides cloud browser sandboxes (connected via CDP protocol) during evaluation. For local debugging, you can use tools like Playwright on your own.

Q: 可以使用哪些模型？

Q: What models can I use?

A: 不限。你可以使用任何模型或多模型组合，包括但不限于 GPT-4o、Claude、Gemini、Qwen、开源模型自部署等。唯一要求是通过 OpenAI 兼容 API 接入。

A: No restrictions. You can use any model or combination of models, including but not limited to GPT-4o, Claude, Gemini, Qwen, self-hosted open-source models, etc. The only requirement is to connect via an OpenAI-compatible API.
