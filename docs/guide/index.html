如何开始备赛

How to Start Preparing

报名完成后即可进入备赛。建议按以下顺序进行：

You can start preparing immediately after registration. We recommend the following order:

① 阅读论文，理解评测协议

① Read the Paper & Understand the Evaluation Protocols

WebRetriever 论文定义了三种评测协议。本次比赛采用 Protocol III（端到端任务协议）——Agent 需要在真实网站上自主完成完整任务链，最终提取并返回目标信息。建议重点阅读论文中 Protocol III 的相关章节，理解任务要求和评分标准。

The WebRetriever paper defines three evaluation protocols. This competition uses Protocol III (End-to-End Task Protocol) — Agents must autonomously complete full task chains on live websites and extract target information. We recommend focusing on the Protocol III sections of the paper to understand task requirements and scoring criteria.

② 了解数据集

② Explore the Dataset

在 Hugging Face 上浏览 WebRetriever 数据集，熟悉三种协议的任务格式和数据结构。这有助于理解比赛任务的输入输出形式。

Browse the WebRetriever dataset on Hugging Face to familiarize yourself with the task formats and data structures across all three protocols. This will help you understand the input/output format of competition tasks.

③ 基于开源代码构建你的 Agent

③ Build Your Agent from the Open-Source Code

我们的比赛代码模板在很大程度上基于开源代码仓库设计。因此，建议直接从开源项目入手开发你的 Agent：

Our competition code template is largely designed around the open-source repository. So we recommend starting your Agent development directly from the open-source project:

  • Clone 开源仓库，运行示例，理解 Agent 的执行框架
  • 在此基础上改进 Agent 的决策逻辑、提示工程、多步推理策略等
  • 备赛期获得队伍仓库后，只需将你的代码迁移过去即可
  • Clone the open-source repository, run examples, and understand the Agent execution framework
  • Improve the Agent's decision logic, prompt engineering, multi-step reasoning strategies, etc.
  • When you receive your team repository, simply migrate your code over

💡 关键信息：比赛代码模板中，任务文件路径、结果输出目录、浏览器沙箱地址（CDP URLs）均由评测系统通过命令行参数传入，模板已处理好接口。你只需在 config.json 中配好自己的模型 API（Key、Base URL），其他专注于 Agent 逻辑开发即可。

💡 Key Info: In the competition template, the task file path, result output directory, and browser sandbox addresses (CDP URLs) are all passed in via command-line arguments by the evaluation system — the template handles the interface. You only need to configure your model API (Key, Base URL) in config.json, and focus on developing your Agent logic.

评测流程

Evaluation Process

获得队伍仓库后，通过以下流程提交评测：

Once you have your team repository, submit for evaluation through the following process:

Push 代码到 main 分支

将你的 Agent 代码推送到队伍私有仓库的 main 分支。

通过赛事 Bot 发起评测

在 Octo 赛事空间中向 Bot 发送提交指令。

系统自动执行

拉取代码 → 静态检查 → 安装依赖 → 在云端浏览器沙箱中运行你的 Agent 完成 100 道评测任务。

获取评分

评测完成后自动返回成绩。评测窗口期内可多次提交，取最优成绩。

Push code to the main branch

Push your Agent code to the main branch of your team's private repository.

Submit via the Competition Bot

Send the submission command to the Bot in the Octo competition space.

Automated execution

Pull code → static checks → install dependencies → run your Agent in cloud browser sandboxes across 100 evaluation tasks.

Get your score

Results are returned automatically. Multiple submissions allowed within the evaluation window; best score counts.

常见问题

FAQ

Q: 比赛的题目类型是什么？

A: 比赛采用论文中的 Protocol III（端到端任务协议）——要求 Agent 不仅导航到正确页面，还需从页面中精准提取目标信息并返回最终答案。

Q: What type of tasks are used in the competition?

A: The competition uses Protocol III (End-to-End Task Protocol) from the paper — requiring Agents not only to navigate to the correct page, but also to precisely extract target information and return the final answer.

⚠️ 注意：比赛使用的 100 道题目是专门为本次挑战赛全新设计的，与论文中开源数据集里 Protocol III 的 100 道题目完全不同。选手不能通过刷开源题目来获得优势。

⚠️ Note: The 100 competition tasks are entirely new, specifically designed for this challenge, and are completely different from the 100 Protocol III tasks in the open-source dataset. Practicing on open-source tasks will not give you a direct advantage.

Q: 比赛有多少道题目？

A: 共 100 道端到端评测任务，覆盖多行业真实网站场景。

Q: How many tasks are there?

A: 100 end-to-end evaluation tasks covering real website scenarios across multiple industries.

Q: 需要自己准备 GPU / 推理服务器吗？

A: 是的。参赛 Agent 通过 OpenAI 兼容接口调用模型，选手需自行管理推理资源。可以使用商业 API（如 OpenAI、Claude 等）或自部署开源模型。

Q: Do I need my own GPU / inference server?

A: Yes. Agents call models via OpenAI-compatible APIs. Teams manage their own inference resources — commercial APIs (OpenAI, Claude, etc.) or self-hosted open-source models are both fine.

Q: 浏览器环境需要自己搭建吗？

A: 不需要。评测时由平台提供云端浏览器沙箱（通过 CDP 协议连接）。本地调试时可自行使用 Playwright 等工具。

Q: Do I need to set up the browser environment?

A: No. Cloud browser sandboxes (connected via CDP) are provided during evaluation. For local debugging, you can use tools like Playwright.

Q: 可以使用哪些模型？

A: 不限。你可以使用任何模型或多模型组合（GPT-4o、Claude、Gemini、Qwen、开源模型等），唯一要求是通过 OpenAI 兼容 API 接入。

Q: What models can I use?

A: No restrictions. Any model or combination (GPT-4o, Claude, Gemini, Qwen, open-source models, etc.) is allowed, as long as it connects via an OpenAI-compatible API.

Q: 开源项目和比赛代码模板是什么关系？

A: 比赛代码模板是开源项目的精简版，保留了 Agent 核心执行框架，并预设好了评测系统的接口（任务输入、结果输出、沙箱连接）。你在开源项目上开发的 Agent 逻辑可以直接迁移到比赛模板中使用。

Q: What's the relationship between the open-source project and the competition template?

A: The competition template is a streamlined version of the open-source project. It retains the core Agent execution framework and pre-configures the evaluation system interfaces (task input, result output, sandbox connection). Agent logic you develop on the open-source project can be directly migrated into the competition template.
