参赛流程

How It Works

1
报名 & 组队

2

提交 & 自动评测

如何备赛

How to Prepare

报名后即可开始备赛：

Start preparing right after registration:

① 阅读论文，了解 Protocol III — WebRetriever 定义了三种评测协议，本次比赛采用 Protocol III（端到端任务协议）。建议结合 论文 和 数据集 了解任务格式与评分标准。

① Read the paper & understand Protocol III — WebRetriever defines three evaluation protocols; this competition uses Protocol III (End-to-End Task Protocol). Read the paper and explore the dataset to understand the task format and scoring criteria.

② 基于开源代码构建 Agent — 比赛代码模板基于 开源仓库 设计，建议直接从开源项目入手：Clone 仓库 → 运行示例 → 设计你自己的 Agent。获得队伍仓库后将代码迁移过去即可。

💡 注意：开源项目提供的是 Protocol I（导航协议）的示例实现，相比本次比赛采用的 Protocol III，不包含信息提取部分。由于信息提取的实现方式多样，且与所设计的 Agent 架构密切相关，需要参赛者自行设计与实现。

💡 Note: The open-source project provides a sample implementation for Protocol I (Navigation Protocol). Compared to Protocol III used in this competition, it does not include the information extraction component. Since extraction approaches vary widely and are closely tied to your Agent design, participants need to implement this part on their own.

📂 开源项目提供了一个基于 UI-TARS 1.5 的完整 Agent 实现示例，核心代码在 src/agent/ 目录下，包括 Agent 交互逻辑、浏览器控制、Prompt 模板等模块。详见 GitHub 仓库。

📂 The open-source project provides a complete Agent implementation example based on UI-TARS 1.5. Core code is in src/agent/, covering Agent interaction logic, browser control, and prompt templates. See the GitHub repository for details.

② Build your Agent from the open-source code — The competition template is built on the open-source repository. Start there: clone → run examples → design your own Agent. Migrate your code when you receive your team repository.

常见问题

FAQ

Q: 比赛题目是什么类型？有多少道？

A: 共 100 道基于 Protocol III 的端到端评测任务，覆盖多行业真实网站。Agent 需从指定网页出发，自主导航并提取目标信息返回最终答案。

Q: What type of tasks and how many?

A: 100 end-to-end tasks based on Protocol III, covering real websites across multiple industries. Agents must navigate from a designated webpage and extract target information to return the final answer.

⚠️ 比赛题目均为本次挑战赛独立设计，与开源数据集中的评测题目互不重叠。

⚠️ All competition tasks are exclusively designed for this challenge and do not overlap with any tasks in the open-source dataset.

Q: 需要自己准备推理资源吗？

A: 是的。Agent 通过 OpenAI 兼容 API 调用模型，选手自行管理推理资源——商业 API（OpenAI、Claude 等）或自部署开源模型均可。

Q: Do I need my own inference resources?

A: Yes. Agents call models via OpenAI-compatible APIs. Teams manage their own resources — commercial APIs (OpenAI, Claude, etc.) or self-hosted models are both fine.

Q: 浏览器环境需要自己搭建吗？

A: 不需要。评测时平台提供云端浏览器沙箱（CDP 协议连接）。本地调试可自行使用 Playwright 等工具。

Q: Do I need to set up the browser environment?

A: No. Cloud browser sandboxes (via CDP) are provided during evaluation. For local debugging, use tools like Playwright.

Q: 可以使用哪些模型？

A: 不限。任何模型或多模型组合均可（GPT-4o、Claude、Gemini、Qwen、开源模型等），唯一要求是通过 OpenAI 兼容 API 接入。

Q: What models can I use?

A: No restrictions. Any model or combination is allowed, as long as it connects via an OpenAI-compatible API.

Q: 并发数、超时和步骤限制是多少？

A: 最高 8 个任务并发；单次模型请求超时 3 分钟；每个任务最多 100 步；每个任务无重试，失败即 0 分。

Q: What are the concurrency, timeout, and step limits?

A: Up to 8 concurrent tasks; 3-minute timeout per model request; max 100 steps per task; no retries — failure scores 0.

Q: 是否允许使用外部搜索引擎？

A: 不允许。Protocol III 考察从指定网页出发的导航与信息提取能力。赛后将进行操作轨迹验证，确认结果通过浏览器操作获取而非搜索引擎。

Q: Are external search engines allowed?

A: No. Protocol III evaluates navigation and extraction from a designated webpage. Post-competition trajectory verification ensures results were obtained via browser operations, not search engines.

Q: 开源项目和比赛模板是什么关系？

A: 比赛模板是开源项目的精简版，保留核心 Agent 框架并预设了评测接口。你在开源项目上开发的 Agent 只需简单适配即可迁移到比赛模板中使用。

Q: What's the relationship between the open-source project and the template?

A: The template is a streamlined version of the open-source project with pre-configured evaluation interfaces. Agent logic developed on the open-source project can be directly migrated.
